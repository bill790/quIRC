quIRC: Tutorial

quIRC is a lightweight terminal-based IRC client. It is written in C.

Foreword

I like to engage with my user population. I welcome bug reports, feature requests, patches, and even undirected rambling musings about what direction quIRC should take in the future. If you use (or are considering using) quIRC, please drop into #quirc on irc.newnet.net for a chat.

This is a tutorial for new users. There is also a reference manual.

Internet Relay Chat

IRC is an Internet chat protocol. It is mainly designed for group communication in discussion forums, called channels, but also allows one-to-one communication via private message. For an overview see the Wikipedia article on IRC. There is a good IRC primer for newbies on www.irchelp.org.

There are several IRC networks, each of which consists of a tree of servers to which users ('clients') connect. Each user on a given network is identified by a nick, which may be their name or a handle. Users can communicate through private messaging, but most conversations take place in channels. A channel is analogous to a room; a line typed into a channel by one user is seen by all other users in that channel.

Channels are policed by chanops (channel operators), who have the power to kick clients out of a room or even ban them. Different channels will have different rules on what is acceptable.

Getting Started with quIRC

Assuming your distributor hasn't packaged quIRC, the first thing you will need to do is compile it. After downloading the source tarball (or cloning the git repository) run make; sudo make install. This should compile quIRC and install it under /usr/local; to install somewhere else (eg. if /usr/local is not in your $PATH), you will need to change the definition of PREFIX in the Makefile. One possible problem at this stage is that your libc may not support asynchronous DNS lookups (getaddrinfo_a()), in which case you will need to edit osconf.h, changing ASYNCH_NL from 1 to 0, and remove -lanl from the OPTLIBS definition in the Makefile. Also, if your C compiler is not gcc, you will need to change CC (and probably also CFLAGS and OPTFLAGS) in the Makefile.

If all of this sounds complicated, and you are running Linux on x86, you can just download the binary tarball (though you will still need to sudo make install). Alternatively, OpenSUSE and SLE users can use the package from the openSUSE Build Service.

Once you've installed quIRC, you will want to connect to a server and join a channel. Good places to start are #quirc on NewNet, #newbies on EFnet, or #irchelp on ColdFront. We'll walk you through connecting to #quirc.

First, choose and set your nickname; you can use your real name or any handle you like to go by. Try not to pick anything too common, like 'john', as it's likely to already be in use. Set your nickname with /nick yournickhere.

Next, connect to the NewNet network, with /server irc.newnet.net. A new tab will open, and various information, including the MOTD, will scroll by. If you get a message like "foobar: Nickname is in use", choose a different nick.

Now you're ready to join the channel; type /join #quirc. Another new tab will open (you can switch between tabs at any time with Ctrl-left and Ctrl-right or by typing /tab n or pressing the Fn key to switch to tab n) and the channel topic will be displayed.

Any messages you type into this tab will be sent to all the other users on the channel. Say hello!

You might find that you don't get an answer straight away, particularly if the channel isn't very busy. It's a good idea to be patient.
